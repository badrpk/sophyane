# Sophyane 11

Sophyane 11 introduces verified autonomous execution for supported software workflows.

## Core execution graph

Supported builds now follow a stateful workflow:

```text
plan
→ build
→ test
→ repair on failure
→ rerun tests
→ verify acceptance criteria
→ write an evidence report
```

## Verified capabilities

- Explicit acceptance criteria before execution
- Real project and file creation
- Exact test command capture
- Standard output, standard error, and exit-code capture
- Bounded repair loops with a safety step limit
- Verification before success claims
- Machine-readable `benchmark_report.json` evidence
- Correct backend routing without `index.html` fallback
- Android Termux compatibility without mandatory runtime dependencies
- Persistent memory, provider plugins, safe tools, diagnostics, CLI, and web UI

## Inventory REST API reference workflow

The first production workflow recognizes requests containing inventory, REST API, SQLite, and automated tests. It creates and verifies:

- `app.py`
- `test_app.py`
- `README.md`
- `inventory.db`
- `benchmark_report.json`

The generated API includes health, create, list, retrieve, update, and delete operations. Tests cover repository and HTTP behavior.

## Evidence contract

Sophyane 11 does not report completion unless all mandatory checks pass. A successful report includes:

- acceptance criteria
- created files
- exact test command
- test exit code
- test output
- per-check PASS/FAIL status
- repair-attempt count
- project and report paths
- final verified result

## Scope

Version 11 establishes the autonomous state-graph architecture and a fully verified reference workflow. It does not yet claim support for every arbitrary software project or parity with every LangGraph/LangChain integration. Additional workflow templates can be added without changing the evidence and verification contract.
